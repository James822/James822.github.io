Introduction to Group Theory and Permutations
2025 Mar 09, James Spratt

I've recently taken a course in group theory, a branch of abstract algebra that studies the mathematical structures known as groups.

Abstract algebra is a fascinating study in mathematics, as it unifies concepts previously scattered in different areas. Boolean algbera for example is the mathematical structure that underlies working with operations on any kind of binary system. Propositional logic, digital circuit design (logic gates), and even set operations are all forms of boolean algebra. Set operations, propositonal logic, and the design of combinational circuits may seem disparate; but their structure is essentially the same.

This is why I have come to love abstract algebra as a discpline. At it's core, it is the formal study of mathematical essence. Two structures may have different substance, but if they function exactly the same up to some kind of isomorphism; then they are the essentially the same.

Groups, the primary study of group theory, happen to be a type of structure that is encountered very frequently in mathematics. Many other structures such as vector spaces, rings, fields, the real numbers; can all be shown to have group-like structures.

Group theory is quite a vast topic, so I'm only going to focus on the study of permutation groups in this post, but this is not without justification. According to Cayley's Theorem, every group is isomorphic to some permutation group. This is another way of saying all groups can be understood in terms of permutations, so it might not be such a bad idea to start there.


Permutations

A permutation of a set of numbers {1,2,...,n} is a bijection from {1,2,...,n} to itself. So it is a function of the form:

τ:{1,2,...,n}→{1,2,...,n}

As an example, let us define τ with n=3 to be the following function:

τ:{1,2,3}→{1,2,3}

Given by: τ(1)=2,τ(2)=3,τ(3)=1.

This means simply that 1 is sent to 2, 2 is sent to 3, and 3 is sent to 1.

Rather than painstakingly write out the function value for each input, we can represent this same exact function in τ in tabular form as the following:

[123231]

This format is known as Cauchy's two-line notation, but we will refer to it as tabular form. All entries are read "top-down" per column. In any given column, the element of the top row is "sent" to the element of the bottom row.

More generally, we can represent any permutation with the following form:

[123...na1a2a3...an]

And this is read as 1 goes to a1, 2 goes to a2, 3 goes to a3, and so on. Here, n is the number of elements to be permuted, and ai are the numbers belonging to the set {1,2,...,n}, and it also must be the case that ai≠aj iff i≠j. Basically, there can't be any duplicate ai. Any kind of table that follows this specific format is a valid permutation.

Notice also that the order of columns does not matter, as we could freely swap them to our liking without changing the permutation. As a matter of style, we typically like to order the columns by the first/top row with 1,2,3,... etc.


Compositions of Permutations

Since permutations are bijective functions from a set to itself, it is possible to compose any number of permutations we like, as long as the permutations are both of n elements.

The operation of function composition is given by σ∘τ(a)=σ(τ(a)) for two permutations τ and σ.

For example, if we take two permutations such as these:

τ=[123213],σ=[123231]

The composition of σ and τ is σ∘τ, which means we apply the permutation τ first and then we apply σ second. Composition is read from "right-to-left" which can be confusing.

Let's see where the number "1" goes under the composition of σ∘τ: τ(1)=2, which implies that σ∘τ(1)=σ(τ(1))=σ(2), and therefore: σ(τ(1))=3. A similar process can be performed for "2" and "3" to show that: σ(τ(2))=2, and σ(τ(3))=1.

There is a much easier way to create compositions of permutations by using the tabular form directly. Take the first permutation τ and superimpose it on top of the second permutation τ. (order matters here, don't put σ on the top and τ on the bottom). You should have something like this:

τ:[123213] σ:[123231]

Then, rearrange the columns of σ so that the numbers at the top row of σ match the numbers of the bottom row of τ. In this example, all we need to do is swap column 1 and column 2 of σ just like so:

[123213] [213321]

We've changed the arrangment of the columns in σ, but as we discussed before this does not change the permutation.

The next step is to simply remove the bottom row of τ and remove the top row of σ. Basically, remove the middle two rows and leave the very top and very bottom row of the whole thing:

[123321]

But wait, doesn't this just look like another permutation? And in fact it is! Compositions of permutations are themselves permutations. This may seem trivial, but it is an important fact that we will come back to later.


Cycle notation

Another way to represent permutations is with something called cycle notation.

A cycle is a permutation that takes a collection of distinct numbers in a cycle, leaving all other numbers fixed. For example: 4 -> 2, 2 -> 6, 6 -> 3, 3 -> 4; is a valid cycle because it wraps back around and no number is visited more than once (besides of course the starting number 4, which must be visited twice). The other numbers are left fixed and unchanged.

We can represent this example with the following cycle notation:

(4263)

Each number sends you to the next one, and once you reach the end of the list, it wraps back around to the first element. This example is a 4-cycle as there are 4 distinct numbers in the cycle. It also doesn't matter where we start the list as long as we maintain the order, so we could also represent the same 4-cycle as so:

(2634)

and you can verify that this changes nothing, but we tend to prefer to start cycles with the least element as a matter of style. Since any k-cycle is also a permutation, we can also represent it using our tabular form like before. The numbers 1 and 5 are not in this cycle which means that they stay fixed as we mentioned before, so we can write:

(2634)=[1234516425]

But can any permutation be represented by a k-cycle? Let's investigate this to see if it is true by considering the following permutation:

[1234521453]

Could we represent this as a single cycle? The answer is no! You can try as hard as you want but there won't be a single k-cycle that can represent this permutation. It turns out however that we can represent this as a composition of 2 cycles, specifically like this:

[1234521453]=(12)(345)

Cycle composition follows the exact same rule as permutation composition like we discussed before. Don't forget that a cycle is a permutation, just a different way of representing it. First, apply the permutation of the first cycle (345), and then you apply the permutation of the second cycle: (12) (recall that permutation composition is right-to-left). The cycles here are also disjoint as they have no elements in common.

It can be proved that every permutation can be represented either as a single cycle, or as a composition of 2 or more disjoint cycles. It is also the case that the order of compositions doesn't matter with disjoint cycles, so the following equality holds for the previous example:

(12)(345)=(345)(12)

The idea is that if a composition of any number of cycles are disjoint, then there are no elements in common. Any element will only be "permuted" by exactly one of the cycles that happens to contain that element. The other cycles will leave that element fixed, so therefore it doesn't really matter where we place the cycle in the composition.

Also, if we wanted to, we could for example rewrite the cycle (375) as:

(1)(2)(375)(4)(6)

but writing all of these extra 1-cycles is redundant, so we usually leave them out.

If we want to write the identity permutation (that is, the permutation that takes every number to itself), then we can simply write it like so with empty brackets (). This element is special, and so we give it the letter e (not to be confused with euler's number):

e=()=[123...n123...n]
Converting Between Cycle Form and Tabular Form

There is a straightfoward way to convert from cycle-form to tabular-form so let's take an example composition of disjoint cycles and see how we can convert it to tabular form:

(15)(246)(79)

Since the largest number here is 9, that means that we are dealing with 9 numbers to be permuted, so create an empty table that has numbers 1 through 9 on the top, and leave the bottom row empty:

[123456789.........]

Our goal here is to "fill in the dots" and determine what numbers need to go there. For starters, we observe that the numbers 3 and 8 are not in the cycles whatsoever, so we know that they stay fixed:

[123456789..3....8.]

Then, we start at the number 1 and see where it maps to. Since the only cycle that 1 belongs to is: (15), we know that 1 maps to 5:

[1234567895.3....8.]

Similarly, we can see that 2 maps to 4 because of the cycle (246):

[123456789543....8.]

And if we continue this process we get for the remaining numbers, then we see that we get the following permutation in tabular form:

[123456789543612987]

And you can verify that this is correct.

We also can do the opposite and convert from tabular form to disjoint cycle-form by simply picking numbers and seeing what cycles they belong to. Let's take this same example and convert it back to cycle-form. To start, we pick the number 1 and "follow" it until we get back to 1. In this case, 1 goes to 5, and 5 goes back to 1, so we write that cycle out:

(15)

Then, we pick the next number that has not been visited yet. So we pick 2, and see where that goes. 2 goes to 4, 4 goes to 6, 6 goes to 2; so that should give us (246). Now add this cycle to the existing cycles like so:

(15)(246)

And then we move to the next number that is not already in a cycle, which is 3; and we see that 3 goes to itself, so we can leave it out.

Then the next number that we haven't visited yet is 7. If we follow 7 then we see that 7 goes to 9 and then goes back to 7, giving us:

(15)(246)(79)

Finally, the last number we need to visit is 8, but 8 just maps back to itself so we leave it as is.

Using cycle notation is generally preferred over tabular form as it takes up less space, and it also better represent a core feature of the permutation in a better way than tabular form does. Using disjoint cycles is really handy so we will continue with this format primarly, but feel free to use tabular form as well to help you understand cycle form.


Inverses and The Identity Permutation

Every permutation has an inverse permutation. For every permutation σ, there exists an identity permutation which we denote σ−1, such that the following holds:

σ−1∘σ=σ∘σ−1=e

The proof that every permutation has an inverse follows from the fact that permutations are bijective functions from a set to itself. Every bijective function has an inverse, and since permutations are from a set to itself; then that inverse is also a permutation.

Let's take the following permutation σ as an example:

σ=(357)(14)

The inverse of this permutation is σ−1=(537)(14) as:

(357)(14)(537)(14)=e

Recall that cycle composition is from right-to-left. Take every from 1 to 7 and move it from right to left across these 4 cycles, and you will see that the number ends up back at itself.

For example: 1 goes to 4 in (14), 4 stays fixed in (537), 4 goes to 1 in (14), and 1 stays fixed in (357). So, 1 goes to 1, itself. The same is true for every other number.

If we go back to using tabular form, then there is similar "trick" to finding out what the inverse of any permutation is. First, let's take this same example and convert it to tabular form:

σ=(537)(14)=[12345674271365]

To find the inverse In tabular form, simply swap the top row and the bottom row of σ which will give you the inverse σ−1:

σ−1=[12345674271365]−1=[42713651234567]

If you want, you can then swap the columns around so that the top rows is ordered from 1,2,3,... and so on, just like this:

σ−1=[42713651234567]=[12345674251763]

You can verify that this indeed does give you the inverse.

The inverse of an inverse is itself, i.e. (σ−1)−1=σ, and this is true for any function but you can verify for this special case of permutation functions.


Extended Compositions

An interesting thing we can do is compose a permutation with itself, i.e. σ∘σ, which leads to some interesting things. For one, we can write as a shorthand: σ∘σ=σ2, understanding that the superscript here refers to composition and NOT exponentiation.

We can generalize this to any integer n>0 so that:

σn=σ∘σ∘...∘σ⏟n times

We can extend to the number 0 by defining:

σ0=e

And, we can also extend into the negative integers by defining the following:

σ−n=(σ−1)n=σ−1∘σ−1∘...∘σ−1⏟n times

where n>0.

With this cool notation, we have found a way to use any integers, negative or non-negative so that the following expression:

σr∘σk

is a well-defined composition of compositions for any integers k, r.

Group Theory Axioms

All of this might seem very trivial to talk about, but consider a game that has 20 loading zones (which is not uncommon, and in fact many games have more), then the amount of permutations of 20 is 20 factorial, which is over 2 quintillion permutations, which is approximately 2 million terabytes of information! The growth of the factorial function is very bad, and it would be absurd to brute force this problem.

Of course, most randomization algorithms will just pick permutations at random, but then they might not have certain properties that we would like. The naive approach would be to generate every permutation and search the permutations to find the desirable ones, but our results from group theory (especially k-cycles) makes it easy to select from literally billions of billions of combinations!
